# It's Thriller Time (B42 整活MOD)

当丧尸靠近玩家或玩家进入战斗状态时，将有概率触发 Thriller 特有的舞台 事件！
当触发舞台时间时， 穿着MJ的经典MV套装的丧尸将带着一小队伴舞登场。
如果安装了authentic Z， 主演与伴舞们则会有不同的服装！ 如果没有，则是默认的原版服装：领队为着色西装套，伴舞则为spiffo玩偶装。
如果安装了True Action Dance，主演与伴舞们则会一边移动一边跳舞，来到玩家面前！

主演丧尸将获得超强化buff - 记忆、力量、体质调到最高档，HP巨量增强。
伴舞丧尸则获得强化buff - 力量、体质为最高档，HP大量增福。

注意：伴舞存活期间，主演接近无敌并且会给伴舞们加血

当然也有另一个模式：在我的BGM中谁也无法打到我！
将沙盒设置切换为BGM模式时，则不会触发 Thriller 舞台事件。
反之每次被丧尸包围时，都将会是玩家独特的个人秀！

玩家在BGM播放的前半时间内，恐慌与压力将暂时恢复，并获得移动与战斗速度的双重增福！
顺带一提，歌曲文件可以自行替换成喜欢的曲目，作为个人专属战斗BGM。

## 机制说明

### 检测机制
每2秒根据玩家所在地点检测一次丧尸距离，
当玩家确实被多个丧尸包围的时候，
根据玩家设置触发 我的BGM 或 Thriller舞台 事件。

当游玩模式为 谁也无法在我的BGM中打到我 时，只要事件冷却期过了并且被多个丧尸包围，就一定会触发 个人秀事件。 MJ的歌曲作为战歌响起，然后获得持续1真实分钟的增益。

### 事件周期
根据MOD设置的事件周期, 也就是``每2秒的丧尸扫描，每分钟的状态判定，以及玩家的攻击丧尸瞬间触发的战斗信号`` ，进行一次逻辑判断处理。

所有范围较大的循环处理尽可能压制在最低消耗，减少大型模组包环境下的运行压力。

### 丧尸变速与状态增幅
通过B42提供的新接口直接``doFasterShambler()``,``doSprinter()``等。
由于力量与体质暂时无可用设置方法，暂时用全局沙盒调整。
全局沙盒调整只会影响本MOD运行时，在召集群演过程中被选中的丧尸，不会影响世界上已经存在的丧尸。

### 事件机制
舞台事件确认触发时，玩家ModData中会缓存触发坐标。并广播到服务器上。
全服只会有一个Thriller舞台事件会确认进行，同时向所有玩家附近刷出群演丧尸与伴舞丧尸，但舞台中心只会存在于触发的玩家周围，并确保全服主演丧尸只存在一个。

要注意的事，当有伴舞存活时，主演接近无敌，需尽快消灭伴舞们。
如果没达成提前结束事件的目标，也就是
``活跃玩家们的可感知范围，也就是30米范围内已经没有任何一只丧尸``
``主演与伴舞都确认死亡或实例被其他MOD消除，且 [现实5分钟在不同日长下对应的游戏分钟时间] 内没有人被任何丧尸盯上``
``单人模式下，处于安全状态持续 [现实10分钟在不同日长下对应的游戏分钟时间]``
``多人模式下，全体进入安全区并都逗留满 [现实20分钟在不同日长下对应的游戏分钟时间] ``
等条件，
当音乐播放满5（沙盒设置数，也就是最大波次）首歌时，舞台事件将自动进入尾声状态，也就是60游戏分钟的淡出周期。
每首歌开始播放时，都会添加一小波丧尸（沙盒设置的群演项目），依照前奏phase(与最大波次一致)进入各玩家范围。
淡出时间过后，事件会彻底结束，并将主要演员们（如果还存在）从世界上彻底删除。

当事件结束时，如果主演已经死亡，在主演消失前，现场将会有个奖励包裹掉落。

BGM模式时，没有额外机制，自然也没有奖励。
但每个人都会有自己专属的BGM舞台。
BGM舞台将随音乐播放结束时自然结束，如果周围丧尸提早清空或进入了服务器设置的绝对安全区，即使BGM还在流淌，也会提早结束。

### 群演丧尸设置
如果装了authentic z，手上都会有荧光棒（随机颜色）
群演与主演丧尸们都将由 addZombiesInOutfit 方法添加

### 丧尸跳舞原理与战斗机制
利用``zombie:setVariable()`` 设置动画条件，
将丧尸的攻击、移动、寻路等动画设置为特定动画。
并且根据``onTick``事件返回的节拍，切换对应的动画。

当安装了TAD的时候，丧尸会在游戏引擎播放对应动画序列时，会因设置的条件而切换对应的动画

当还未受到攻击时，主演丧尸们会在距离玩家3~4格的可靠位置跳舞
一旦受到攻击，伴舞们会立刻反击。
而主演则会瞬移到玩家攻击范围外的可靠位置上，一边跳舞一边给伴舞们激励效果（每秒10%血量恢复）
主演们的攻击动画在TAD状态下可能会与其他时候不同。

### 音乐机制
音乐播放并不会吸引丧尸。
但Thriller舞台触发时，将会在玩家身上挂在几秒的``addSound()``，并将包围圈内的丧尸的对象设置为玩家。

舞台事件结束但音乐还未完全播放完毕时，会渐变淡出
舞台进入淡出状态但音乐已经播放完毕时，不会继续播放。

当沙盒选项只选一首歌曲时，只会播对应的曲子。
当沙盒选项全部留空时，如果通过别的MOD添加了战斗BGM，则会播放添加的战斗BGM。
当沙盒选项全选时，则会播放全部曲子，也包括额外追加的BGM。

### 丧尸控制机制
主要通过``onTick``事件执行``toPathLocation`` 微调丧尸所在位置与行为。
尽可能将处理逻辑收缩在不影响性能的范围。

距离玩家剩下3格距离时会停止强制控制。
当距离玩家大约7格时会重启指路事件。

### 事件状态机与触发钩子

主要由 ``Stage`` 模块管理。通过``StageMin``每游戏分钟确认各个flag的状态，
根据状态分发到不同的管理模块中。

BGM模式与舞台模式共享一个状态机与触发器。
只是BGM模式不需要复杂的判定，只需要确认玩家是否被丧尸包围，同时已经过了BGM增益冷却期。
以及BGM模式没有luring状态。 确认触发即进入BGM增益状态。
